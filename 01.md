# ✅ Data Selection & Filtering in Pandas 🚀
> This is one of the most used skills in real ML workflows.
> You will select features, filter bad rows, create subsets for train/test, remove outliers — almost every day!

---
> We continue with the same realistic dataset:
```python
import pandas as pd
```
```
data = {
    "Name": ["Aman", "Riya", "John", "Priya", "Karan", "Rohit", "Anita"],
    "Age": [22, 21, 23, 24, 22, 25, 21],
    "Salary": [25000, 27000, 30000, 32000, 25000, 35000, 27000],
    "Department": ["IT", "HR", "IT", "Finance", "IT", "Finance", "HR"],
    "Experience": [1, 2, 2, 3, 1, 4, 2]
}
```
> df = pd.DataFrame(data)

# 1️⃣ Selecting Columns
## Single column → returns Series
```
df['Name']
```
---
```
Output:
0     Aman
1     Riya
2     John
3    Priya
4    Karan
5    Rohit
6    Anita
Name: Name, dtype: object
```
## Multiple columns → returns DataFrame
```
df[['Name', 'Age', 'Salary']]
```
## Output:
```
Name  Age  Salary
0    Aman   22   25000
1    Riya   21   27000
2    John   23   30000
3   Priya   24   32000
4   Karan   22   25000
5   Rohit   25   35000
6   Anita   21   27000
```
---
## 🔥 ML Use
> Feature selection (keep only useful columns)
Drop unnecessary columns before training
Reduce memory & training time
2️⃣ Selecting Rows (Basic Slicing)
df[0:3]          # First 3 rows (0 inclusive, 3 exclusive)
df[2:5]          # Rows 2, 3, 4
Output of df[0:3]:
Name  Age  Salary Department  Experience
0   Aman   22   25000         IT           1
1   Riya   21   27000         HR           2
2   John   23   30000         IT           2
🔥 Note: This works but is not recommended for complex filtering → use loc/iloc instead.
3️⃣ .loc[] → Label-based Indexing (Most Powerful)
# Rows 0 to 3 (inclusive), Columns 'Name' to 'Salary' (inclusive)
df.loc[0:3, 'Name':'Salary']
Output:
Name  Age  Salary
0   Aman   22   25000
1   Riya   21   27000
2   John   23   30000
3  Priya   24   32000
# Single cell
df.loc[5, 'Salary']          # → 35000
🔥 ML Use
When your index is dates, IDs, or names
Very readable and safe
4️⃣ .iloc[] → Integer Position-based Indexing
# Rows 0-2, Columns 0-2 (Name, Age, Salary)
df.iloc[0:3, 0:3]
Output: Same as df[0:3] above.
# Specific rows & columns by position
df.iloc[[0, 2, 5], [1, 3]]   # Age and Department of rows 0,2,5
Output:
Age Department
0    22         IT
2    23         IT
5    25    Finance
🔥 Key Difference
.loc → uses labels (inclusive on both sides)
.iloc → uses positions (0-based, exclusive on end like Python slicing)
5️⃣ Boolean Indexing (Filtering) — The Real Game Changer
# Single condition
df[df['Age'] > 22]
Output:
Name  Age  Salary Department  Experience
2    John   23   30000         IT           2
3   Priya   24   32000    Finance           3
5   Rohit   25   35000    Finance           4
# Multiple conditions (AND = &, OR = |, NOT = ~)
df[(df['Age'] > 22) & (df['Department'] == 'IT')]
Output:
Name  Age  Salary Department  Experience
2  John   23   30000         IT           2
# Filter rows + select specific columns (Best Practice)
df.loc[df['Salary'] > 30000, ['Name', 'Salary', 'Department']]
Output:
Name  Salary Department
3  Priya   32000    Finance
5  Rohit   35000    Finance
🔥 ML Use Cases
Remove outliers (df[df['Age'] < 100])
Class balancing
Create training subset (df[df['Experience'] >= 2])
Target variable filtering

🔥 Real ML Interview Question
Interviewer: "How will you select all employees from IT department with salary > 28000?"
Best Answer:
df.loc[
    (df['Department'] == 'IT') & (df['Salary'] > 28000),
    ['Name', 'Salary', 'Experience']
]
(or use df.query('Department == "IT" and Salary > 28000'))
❌ Common Beginner Mistakes
Forgetting parentheses: df[df['Age']>22 & df['Salary']>30000] → Error
Using df['col'][5] instead of df.loc[5, 'col']
Mixing loc and iloc wrongly
Forgetting that boolean indexing keeps original index
